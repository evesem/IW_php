# Инструкции по запуску проекта

1. Установка и запуск
```bash
# Перейти в папку проекта
cd D:\IW\project
```

# Запустить PHP-сервер
```bash
php -S localhost:8000
```
2. Настройка базы данных
```bash
# Войти в MySQL
mysql -u root -p

# Создать БД и таблицы (выполнить в MySQL)
CREATE DATABASE idea_catalog;
USE idea_catalog;

CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('user', 'admin') DEFAULT 'user'
);

CREATE TABLE ideas (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    category VARCHAR(50) NOT NULL,
    priority TINYINT DEFAULT 1,
    user_id INT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

# Создать администратора

```

3. Настройка подключения

В файле config/database.php укажите пароль от MySQL:

```php

```

4. Открыть в браузере
text
http://localhost:8000

# Описание лабораторной работы
## Название работы: Разработка веб-приложения «Каталог идей» на PHP с системой аутентификации и ролевой моделью (пользователь/администратор).

## Краткое описание
В рамках лабораторной работы разработано веб-приложение, позволяющее пользователям создавать, просматривать и искать идеи. Администратор имеет расширенные права для управления пользователями и контентом.

## Основные разделы приложения

| Раздел | Описание |
|---|---|
| Главная страница | Отображает последние идеи из БД (доступна всем) |
| Регистрация / Вход | Аутентификация с хешированием паролей |
| Создание идеи | Форма с 6 полями и валидацией |
| Поиск | Фильтрация по ключевым словам, категории, приоритету |
| Админ-панель | Управление пользователями и идеями (8 функций) |

## Используемые технологии

| Технология | Назначение |
|---|---|
| PHP 8 | Серверная логика |
| MySQL | Хранение данных |
| PDO | Работа с базой данных (защита от SQL-инъекций) |
| Сессии | Управление доступом |
| bcrypt | Хеширование паролей |
| HTML/CSS + Bootstrap | Интерфейс |

## Сценарии взаимодействия пользователей

Сценарий 1: Обычный пользователь

```text
1. Открывает главную → видит идеи
2. Регистрируется → входит
3. Создаёт новую идею (заполняет форму)
4. Ищет идеи по ключевому слову
5. Выходит из системы
```

Сценарий 2: Администратор

```text
1. Входит под логином admin
2. Переходит в админ-панель
3. Создаёт нового администратора
4. Редактирует роль пользователя
5. Удаляет спам-идею
6. Выходит из системы
```

Сценарий 3: Гость (неавторизованный)

```text
1. Заходит на главную
2. Видит только публичные идеи
3. Может искать, но не может создавать
4. Видит кнопки «Вход» и «Регистрация»
```
